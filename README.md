# ⚔️ EffectNoKill

> **PaperMC-плагин для управления уроном, заморозкой и неуязвимостью игроков в реальном времени.**

---

## 📋 Описание

**EffectNoKill** — это легковесный плагин для серверов на базе [PaperMC](https://papermc.io/), предоставляющий администраторам и модераторам полный контроль над состоянием игроков в бою:

- 🛡️ Полная или частичная блокировка входящего урона
- 🧊 Заморозка игроков на месте (с сохранением обзора)
- ⚡ Умножение или деление входящего урона в 2–5 раз
- 🔄 Мгновенный сброс всех эффектов — для одного игрока или сразу для всех

Отлично подходит для мини-игр, ивентов, PvP-арен и административных нужд.

## 🚀 Установка

1. Скачайте или соберите `.jar`-файл плагина (см. раздел [Сборка](#-сборка)).
2. Поместите файл в папку `plugins/` вашего сервера.
3. Перезапустите сервер.

---

## 🏗️ Сборка

Для сборки используется [Gradle](https://gradle.org/). Убедитесь, что установлена Java 21+.

```bash
./gradlew build
```

Готовый `.jar` окажется в папке `build/libs/`.

Для быстрого запуска тестового сервера:

```bash
./gradlew runServer
```

## 🎮 Команды

### `/nokill` — управление заморозкой и блокировкой урона

| Подкоманда | Аргументы | Описание |
|---|---|---|
| `stun` | `<ник>` | Заморозить игрока, убрать урон и откидывание |
| `stundmg` | `<ник>` | Заморозить игрока, но оставить урон |
| `nodmg` | `<ник>` | Убрать урон без заморозки (откидывание сохраняется) |
| `stop` | `<ник>` | Снять все эффекты с конкретного игрока |
| `stopall` | — | Снять все эффекты со всех игроков |

**Примеры:**
```
/nokill stun Steve
/nokill nodmg Alex
/nokill stopall
```

---

### `/upkill` — управление множителем урона

| Подкоманда | Аргументы | Описание |
|---|---|---|
| `updmg` | `<2-5> <ник>` | Увеличить входящий урон игрока в X раз |
| `downdmg` | `<2-5> <ник>` | Уменьшить входящий урон игрока в X раз |
| `stop` | `<ник>` | Вернуть стандартный урон конкретному игроку |
| `stopall` | — | Вернуть стандартный урон всем игрокам |

**Примеры:**
```
/upkill updmg 3 Steve
/upkill downdmg 2 Alex
/upkill stopall
```

## 📦 Структура проекта

```
src/main/java/zov/dayn/effectNoKill/
├── EffectNoKill.java        # Главный класс плагина
├── CombatListener.java      # Обработка событий урона и перемещения
├── NokillCommand.java       # Команда /nokill
├── UpkillCommand.java       # Команда /upkill
├── PlayerStateManager.java  # Менеджер состояний игроков
└── PlayerState.java         # Модель состояния одного игрока
```

---

## ⚙️ Принцип работы

- Каждому игроку соответствует объект **`PlayerState`**, хранящий флаги заморозки, блокировки урона и текущий множитель урона.
- **`PlayerStateManager`** управляет хранилищем состояний и запускает повторяющуюся задачу (каждый тик), которая телепортирует замороженных игроков обратно на точку заморозки.
- **`CombatListener`** перехватывает события `EntityDamageEvent` и `PlayerMoveEvent` с наивысшим приоритетом, применяя нужные модификации в зависимости от состояния игрока.
- При выходе игрока с сервера его состояние автоматически очищается.
